Project Introduction

Background

Synalogik Innovative Solutions Ltd is a UK-based data aggregator supporting government, law enforcement, and national security agencies. It also serves financial institutions, including banks, betting, and insurance organisations.

The organisations flagship platform SCOUT lacked a dashboard, limiting users ability to track task progress or KPIs effectively. Feedback from clients highlighted the need for a clear, interactive dashboard to boost operational insight.

Problem

Without a dashboard, users couldn't easily monitor team performance or task statuses, leading to inefficiencies and frustration. The company lacked a dedicated PM and formal roadmap, so design needed to be swift and user-focused.

The goal was to create a prototype dashboard that would enable users to track KPIs, monitor teams, and gain control over their daily operations. This involved understanding user needs, defining key features, and designing an intuitive interface.

My Roles

I designed and developed the entire dashboard prototype from ideation to clickable model, ensuring it met user needs and business goals.

Solution

Created a modular, interactive dashboard showing task stats, team overview, and actionable insights to help users track KPIs and coordinate work better.

Dashboard Development

Initial Assumptions

Research 1.1

Understanding the Industry Landscape

To design a competitive dashboard, I reviewed existing platforms in the intelligence and SaaS sectors. Most competitors featured dashboards with high-level statistics and team task views to meet analyst and manager needs. These features set user expectations and highlighted gaps to address.

  • High-level metrics drive quick insights: Users rely on summary statistics to gauge overall performance at a glance, enabling faster decision-making.
  • Interactive graphs aid trend spotting: Integrated countdown timers, stock indicators, and limited-time offers to drive conversions.
  • Hero Section Optimisation: Dynamic charts help users explore data trends over time and uncover important patterns.
  • Team task lists improve coordination: Displaying tasks by team members supports transparency and workload management.
  • Customisable views enhance user control: Allowing users to tailor dashboards boosts engagement and relevance.
  • Mobile-friendly designs support remote work: Responsive layouts ensure usability across devices, accommodating modern work styles.

By combining industry best practices with insights from user research, the dashboard was designed to enhance task visibility, improve team collaboration, and increase overall productivity.

Front-End Technology Analysis

Choosing the right front-end framework was critical to balancing cost, development time, and long-term scalability—especially given the organisation's limited front-end development experience. The decision directly influenced the project's budget, timeline, and future adaptability.

  • Alignment with Team Expertise: The framework needed to complement the team’s existing skills in Java and JavaScript, minimising the learning curve and accelerating development.
  • Ease of Adoption: Given the team's limited front-end experience, selecting a user-friendly framework ensured faster implementation and reduced onboarding time.
  • Scalability: The framework had to support future expansion, particularly for iOS (and potentially Android) applications, enabling a unified codebase and reducing maintenance overhead.

This strategic selection streamlined development, optimised resources, and laid a solid foundation for future growth.

Identified Frameworks:

  1. Swift UI (Apple Documentation)
  2. UIKit (Apple Documentation)
  3. React Native (Documentation)
  4. React (Documentation)
  5. MUI (Documentation)

Findings

After evaluating multiple frameworks, it became clear that SwiftUI, while well-suited for iOS development, was not viable due to its incompatibility with the team’s preferred languages. Instead, React Native , React , and MUI emerged as the strongest candidates for the following reasons:

  • Developer Availability & Cost: React has a significantly larger developer pool compared to Swift, with a promising 5:1 ratio, making it a cost-effective long-term investment.
  • Familiar Technology Stack: Built on JavaScript, HTML5, and CSS3, React provides an accessible learning curve for the team, ensuring seamless adoption.
  • Scalability & Maintainability: React’s component-based architecture supports reusable code, reducing development time and enhancing scalability.
  • Extensive Component Library: Pre-built components and icon libraries streamline UI development, ensuring consistency and efficiency.
  • Cross-Platform Compatibility: React Native enables a unified codebase for both iOS and Android, simplifying future mobile app expansion.

User Feedback & Iteration

During the refinement phase, user engagement increased significantly as we tested and aligned key marketing strategies with stakeholder expectations. The Landing and Index pages featured elements such as the Hero Section, USP Strip, and Promotional Banners , which played a crucial role in driving conversions and engagement.

To validate design choices, we presented users with multiple card layout variations tailored for this platform. While feedback was largely positive, further refinements were needed to enhance clarity and usability. These refinements were made using Figma to develop more polished low-fidelity prototypes.

Findings

  • Conversion-Driven Design: Users responded positively to time-limited offers, trust signals, and strategic call-to-action placements, reinforcing the effectiveness of conversion-based design elements.

  • Visual Hierarchy & Engagement: Prominent CTAs in the hero section and curated marketing banners helped guide users seamlessly from discovery to checkout.

  • Psychological Triggers: Features like scarcity indicators (e.g., “Only 3 left in stock”), social proof (e.g., trending product tags), and urgency-based messaging enhanced user motivation.

  • Streamlined Card Layouts: Minimalist card designs with concise information reduced cognitive load, improving readability and click-through rates.

  • Data-Driven Refinements: Insights from user testing led to further refinements in layout structure and interaction flow, ensuring the platform met both business objectives and user expectations.

UX Testing 1.1

System Usability Scale (SUS)

Usability testing is essential for assessing how easily users interact with a product. This project used the System Usability Scale (SUS) as a key metric to measure user satisfaction and usability. The SUS survey, comprising 10 questions, evaluated user perceptions of functionality and ease of use. To enhance efficiency and accuracy, Task-Based and First Click Testing were integrated alongside SUS.

SUS Overview

  • Purpose : Measures overall usability with a 10-question survey.
  • Rating : Responses are rated on a 5-point scale.
  • Score Calculation : Scores range from 0 to 100 based on the answers.

UX Testing: 1.2

Task Design and Question Strategy

Task 1: Navigation

Objective: Assess the ease of navigating the system.

Instructions

  1. Explore the Home page.
  2. Go to the Product List page.
  3. Find and open the Experian product page.
  4. Add the product to your basket.
  5. Confirm your selection in the Added to Basket modal.
  6. Visit Your Basket but do not press Checkout .
  7. Return to the Home page intuitively.

Follow-Up Questions:

  • Ease of Use: How easy was it to find and use the Product List page?
  • Navigation: Was selecting a product and reaching the checkout intuitive?
  • Issues: Did you face any difficulties or frustrations?
  • Improvements: Any suggestions to enhance the shopping experience?

Task 2: Shopping and Form Completion

Objective: Assess platform exploration, form usability, and progressive disclosures.

Task 2

  1. Explore the Home page.
  2. Tap the Experian product banner to open its description page.
  3. Add the product to your basket using the provided options.
  4. Upgrade to Professional and confirm your selection in the Added to Basket modal.
  5. Go to Your Basket and tap Checkout .
  6. Complete the form but do not press Send .
  7. Return intuitively to the Home page.

Follow-Up Questions:

  • Ease of Use: How easy was it to upgrade your product?
  • Form Interaction: Was the form simple or difficult to use?
  • Issues: Did you encounter any problems or frustrations?
  • Improvements: Any suggestions to enhance this prototype?

UX Testing 1.3

Interactive Prototype

This e-commerce prototype, built in Figma™, was designed to optimize user task flow and platform exploration. Tested by 30 participants, it involved two tasks: navigating from the home page to the product list, selecting a product, and choosing a variant on either the product description page or confirmation modal. Success was measured by reaching the basket page without major difficulty.

Results and Analysis

30 participants completed task-based evaluations using the System Usability Scale (SUS), First Click, and Time-Based Tasks, verbalizing their thoughts for deeper insights.

The prototype scored 88 /100 on SUS, surpassing the 68 benchmark, highlighting its strong user-centric design and usability.


Task 1 - Navigation Ease and Efficiency

This test evaluated the usability of the application, from the Home screen to adding an item to the basket and completing the checkout. The task was time-based (6 minutes), with a pre-selected product to simplify the process. The objective was to determine whether navigation was intuitive and straightforward.

  • Results:
  • All participants completed the task successfully.
  • 86% of participants finished within the time limit.
  • 14% took up to 6 minutes to complete the task.
  • No participants encountered navigational issues.

Task 2 - Decision-Making Efficiency

This test assessed the usability of the product upgrade dropdown system, following the same journey as Task 1, with the additional task of upgrading the product before checkout. The dropdown components, positioned logically with dynamic pricing, were placed on the Product Description page.
From a business perspective, the goal was to increase marketing opportunities and incorporate potential decoy items post-analysis. From a user perspective, the system was designed to provide a simple, intuitive, and engaging experience to improve decision-making efficiency.

  • Results:
  • All participants completed the task successfully.
  • 68% engaged with the product upgrade system on the Product Description Page.
  • 22% engaged with the system on the Added to Basket modal.
  • 10% interacted with the system on the Basket Page.
  • The test provided sufficient opportunities for product upgrades before checkout.
